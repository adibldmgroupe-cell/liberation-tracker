Risques péremption : couche décisionnelle (niveau de risque → décisions)

Le niveau de risque (Phase 1) déclenche désormais des décisions recommandées
(partie « Décisions » de la méthodo), auto-déduites du niveau et à confirmer
en Phase 2. Bascule : Faible -> aucun circuit (routine) · Moyen -> validation
Niveau 2 · Élevé -> validation Niveau 3 (Niveau 1 réservé).

- Service : DECISION_MATRIX (faible/moyen/eleve -> forecast, split, validation,
  monitoring) + helpers decisionsFor / needsValidation + libellés. Défauts :
  Moyen = middle / multiple MOQ / Niveau 2 / mensuel ; Élevé = worst / MOQ /
  Niveau 3 / 15 jours ; Faible = best / multiple MOQ / aucun / trimestriel.
  (Pas de migration : décisions dérivées du niveau, confirmées en Phase 2.)
- Page Évaluation : bloc « Décisions » réactif au niveau — Faible affiche
  « produit en routine » ; Moyen/Élevé affiche les 3 décisions + le niveau de
  validation requis + bouton « Lancer le circuit » (désactivé, hook Phase 2).
- Matrice : colonne « Validation » (Niveau 2/3 ou — selon le niveau de risque).
- Thème jour/nuit/atelier.

Testé en preview : Faible (global 1) -> routine ; Moyen -> Niveau 2 (middle /
multiple MOQ / mensuel) ; Élevé (global 5) -> Niveau 3 (worst / MOQ / 15 j).
Pondérations configurables validées en conditions réelles (modif 30/30/40 par
un compte sur le site déployé -> recalcul global 3.2, plusieurs produits évalués).

Phase 2 (circuits de validation Niveau 2 / Niveau 3) : à brancher dès définition
des étapes/services.
